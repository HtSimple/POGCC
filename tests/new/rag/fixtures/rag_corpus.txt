POGCC 项目说明（测试语料）

一、项目背景
POGCC 是 PPT 大纲智能生成与内容补全系统，面向学术汇报与课程答辩场景。
系统要解决 AI 生成 PPT 时常见的逻辑断层与事实幻觉问题。

二、技术路线
核心采用结构化 JSON Schema 协议描述大纲与页面正文。
内容生成链路包括：大纲生成、逐页知识检索、正文扩展、演讲备注生成。
本地 RAG 负责从用户上传资料中检索证据，必要时再触发外部网络搜索。

三、RAG 与事实控制
系统通过 RAG 检索降低事实幻觉：先查本地知识库，再融合检索摘要进入 Prompt。
检索结果应包含来源文件名与文本片段，便于后续事实一致性检查。
质量验证包括 Top-K 检索命中率、来源字段完整性、以及事实准确率评测。

四、性能与测试
性能目标是万字级参考资料入库后，端到端生成流程在合理时间内完成。
测试覆盖文档切分、向量检索、大纲 reference_context 注入、以及检索问答质量。

五、输出形态
最终可导出 Markdown，包含每页标题、要点、正文说明与演讲备注。
历史任务应能回溯大纲、检索结果与生成内容（持久化方案待完善）。

六、扩展说明（用于切分测试）
在大规模课程资料场景下，用户可能上传多章节讲义、实验报告摘要、以及项目答辩稿。
系统需要将长文档按段落与字数上限切分为多个 TextChunk，并保留 source_file 与 chunk_index。
切分后的每个片段应可独立检索，又在合并检索结果时保持主题连贯。
当用户询问「如何降低幻觉」时，应优先召回第三节 RAG 与事实控制相关段落。
当用户询问「JSON Schema」时，应召回第二节技术路线中的结构化协议描述。
当用户询问「Markdown 导出」时，应召回第五节输出形态相关内容。
上述规则用于 RAG 命中率评测：问题与语料段落一一对应，便于计算 Top-K 召回率。
评测脚本读取 rag_queries.json，对每条 query 检查 must_contain_any 与 source_file 字段。
若命中率低于 80%，应检查 embedding 模型、切分粒度或 query 与语料的语义差距。
