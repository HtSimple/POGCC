# PPT 大纲智能生成与内容填充系统（POGCC）流程规约

## 一、用户端操作流程

### 1. 创建生成任务

用户进入系统并填写任务基础信息，参考如下：

- **PPT 主题**
- **使用场景**：如学术汇报、商务汇报、咨询分析等。
- **目标页数**
- **受众对象**
- **额外要求**

### 2. 上传或选择参考资料

- 用户上传 PDF、Word、TXT 文件，或从已有知识库中选择文档。
- **前端反馈**：实时显示解析状态（如“解析中”、“已入库”、“解析失败”）。

### 3. 生成 PPT 大纲

用户点击“生成大纲”，系统返回包含以下要素的结构化大纲，具体参考json规约：

- PPT 标题
- 章节结构
- 每页标题
- 每页核心目标
- 每页初步要点

### 4. 用户修改大纲

用户针对生成的大纲进行人工干预（强化逻辑建模与结构合理性）：

- 编辑页面标题
- 调整章节顺序
- 增删页面或修改页面数量
- 调整每页要点

### 5. 逐页补充知识

用户针对特定页面点击“补充知识”，系统执行：

- **本地 RAG 检索** (内容主要来自用户上传的文档)
- **网络搜索**（必要时触发）
- **结果展示**：展示检索摘要和来源（屏蔽底层分块/向量细节）。

### 6. 生成每页正文内容

根据已确认的大纲、RAG 检索结果、网络搜索结果及 JSON 规约，生成：

- 正文要点
- 扩展说明
- 来源信息

### 7. 生成每页演讲备注

系统自动或根据用户指令生成详细的 Speaker Notes，内容比幻灯片正文更具补充性。

### 8. 事实一致性检查

系统对内容进行自动化审核并给出结果：

- **通过**
- **存在无来源断言**
- **与材料不一致**（建议重新检索或生成）

### 9. 生成最终 Markdown 文本

输出完整报告，包含：

- PPT 标题及每页标题/要点
- 正文说明与演讲备注
- 参考来源及质量检查结果

### 10. 查看历史记录

用户可回溯历史任务，查看历史生成的大纲、内容、备注及最终文本。

## 二、后端处理流程

### 1. 任务持久化

后端创建 `project/task` 记录，存储主题、要求、模型配置及时间戳。

### 2. 文档解析

调用相应解析器处理上传文件：

- **PDF/Word/TXT 解析**：提取纯文本内容。

### 3. 文本清洗与切分

- **预处理**：去除页眉页脚、重复空行。
- **切分**：按语义或长度切分 chunk。
- **元数据标注**：记录来源文件、页码、段落位置。

### 4. 向量化与入库

- 生成 Embedding 并写入向量数据库。
- 业务数据库同步保存文档、分块及任务关联关系。

### 5. 大纲生成与约束

- 基于用户输入和摘要调用 LLM。
- **输出控制**：强制使用 JSON Schema 约束。
- **校验**：使用 Pydantic 或 JSON Schema 进行格式验证。

### 6. 大纲校验与修复

- 若输出不符规约，尝试 JSON 修复或重新调用。
- 多次失败则向前端返回错误。

### 7. 大纲版本管理

保存经过用户修改后的最终版大纲，作为后续生成的唯一基准。

### 8. 逐页知识检索

根据页面上下文构造 Query，执行多级检索：

1. **本地 RAG 检索**
2. **知识完整性评估**
3. **网络搜索补充**（视需求触发）
4. **知识融合**

### 9. 搜索结果处理

- **过滤与融合**：去重、相关度排序、权威来源优先。
- **压缩**：摘要压缩与 Token 长度控制。

### 10. 页面内容生成

基于“大纲 + 证据”生成结构化内容。

### 11. 备注生成

基于页面内容和检索证据，同步或异步生成演讲者备注。

### 12. 事实一致性检查

利用 LLM 问答验证生成内容是否具备检索证据的支持，确保信息准确性。

### 13. 最终 Markdown 组装

按照标准层级结构（# 标题 / ## 页面 / ### 要点、备注、来源）组装文本。

### 14. 成本与 Token 监控

记录每次 LLM 调用的详细数据：

- 模型、Token 使用量（Prompt/Completion）、估算成本、响应耗时、重试记录。

### 15. 质量与性能评价

记录评估指标：

- RAG 命中率、检索相关性、网络搜索有效率、事实一致性通过率、端到端耗时及单任务 Token 成本。
