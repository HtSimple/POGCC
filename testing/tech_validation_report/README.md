# 技术验证报告计划

目标：输出技术验证报告，对比不同输入场景下的内容质量、事实准确性、可用性与生成效率。

## 范围
- 信息深度补全与文案结构化创作
- 输出形式：结构化PPT内容（大纲 + 页面内容 + 备注），以markdown导出
- 模型：DeepSeek与Qwen（若均已配置）
- 模式：本地RAG/网络搜索可开关（按场景需要）

## 工作步骤
1) 定义输入场景
   - 覆盖短主题、强约束、长文档、多文档与混合场景。
   - 参考 scenarios.json。

2) 定义指标与评分标准
   - 内容质量：结构、完整性、一致性、证据使用。
   - 事实准确性：支撑率、错误数、无证据陈述数。
   - 可用性：PPT可用度、编辑成本、受众匹配。
   - 效率：总耗时、单页耗时、Token/成本（可选）。
   - 参考 metrics.md 与 rubric.md。

3) 准备输入材料
   - 文档放在 data/。
   - 记录每个场景所用的文件与参数。

4) 跑生成并保存输出
   - 每个场景分别跑不同模型与模式。
   - 输出保存到 outputs/{scenario_id}/{model}/{mode}/。

5) 评分与记录
   - 按评分标准打分。
   - 事实准确性需对照材料或检索证据验证。
   - 记录到 results/metrics_template.csv。

6) 形成报告
   - 使用 report_template.md 填写结果与结论。

## 建议目录结构
- data/            输入文档
- outputs/         原始输出
- results/         评分表与汇总
- scripts/         可选脚本
