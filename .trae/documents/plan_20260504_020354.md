## 统一配置为 config.json 的改造计划

### 1. 重构 `app/utils/config.py`

* 移除 `python-dotenv` 依赖和 `load_dotenv()` 调用

* 移除 `get()` 中"环境变量优先"的逻辑，改为只从 `config.json` 读取

* 让 `LLMService`、`DeepSeekService`、`QwenService` 统一通过 `Config` 类读取配置，而不是各自 `open("config.json")`

### 2. 改造 LLM 服务使用 Config 类

* `LLMService`：改用 `Config` 类读取 `llm_provider` 和 API Key

* `DeepSeekService`：改用 `Config` 类读取 `deepseek_api_key`

* `QwenService`：改用 `Config` 类读取 `dashscope_api_key`

* 消除三个文件各自重复打开 `config.json` 的代码

### 3. 创建 `config.json.example`

* 提供不含真实密钥的模板文件，方便团队成员复制使用

### 4. 清理 `.env` 相关文件

* 删除 `.env.example`（其中如果有其他配置，需要保留或者等效移植）

* 从 `requirements.txt` 移除 `python-dotenv` 依赖

### 5. 更新 `.gitignore`

* 确认 `config.json` 已在忽略列表（当前已有 ✅）

* 添加 `.env` 到忽略列表

### 6. 更新 `README.md`

* 将环境配置说明从 `.env` 改为 `config.json` 方式

* 更新技术栈说明（移除 python-dotenv）
